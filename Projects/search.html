Search Algorithms

Data Structures and Algorithms
Created a program that uses a command line interface to solve a puzzle entered by the user. The puzzle is a two dimensional map that the user can move around in and press buttons on the floor that will open specific doors, and the goal is to get to the end of the map. The user has multiple options when initiating the executable, as they can select a list or map output, and can request that it be solved with a queue or a stack
Main concepts implemented:
  • Queue and stack data type usage to implement depth first search and breadth first search algorithms
  • Used a deque in order to have both queue and stack implementations in one data type
  • The current state is stored in the game and the simulation keeps track of all reachable states from that state, adding them to a queue or stack. Then it pops the next state and finds all reachable states from there, and so on
  • Implementation of backtracking from finding the goal in order to find an entire path.
  • Use getopt functionality to read in and parse long command line arguments
