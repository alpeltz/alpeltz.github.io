Priority Queue/Stock Simulator

Data Structures and Algorithms
This problem simulated a stock market type environment, using the Star Wars universe as a context. The situation of the program was that there were multiple planets, and each planet would have a number of Jedi and Sith on it. The Jedi could attack the Sith, while the Sith could also ambush the Jedi, and decisions were made based off of high and low power values for each object. This simulates a stock market where decisions are made based off of high and low stock prices. Multiple types of queues were implemented.
Main concepts implemented:
  • Getopt usage for reading long command line arguments
  • Implementation of running medium
  • Implementation of priority queues (unordered, unordered fast, sorted, pairing, binary)
  • Implementation of pairing heap, which consists of nodes with child and sibling pointers
  • Implementation of binary heap that is able to fixUp, and fixDown to ensure the heap is sorted
  • Understanding of abstract data types
