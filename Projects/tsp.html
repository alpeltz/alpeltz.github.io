Traveling Salesman

Data Structures and Algorithms
Implemented multiple different solutions to the traveling salesman problem, optimizing for speed and efficiency. Some algorithms involved using nearest and arbitrary insertion algorithms, implementing two-opt to optimize an existing solution, and using minimum spanning trees in order to create pruning bounds to prune poor partial solutions.
Main concepts implemented:
  • Command line argument interfaces, using getopt_long
  • Implementation of constructing a minimum spanning tree from a set of points
  • Used an arbitrary insertion algorithm to solve TSP
  • Implementation of two-opt to improve a path for TSP
  • Used Perf to analyze run performance and optimization
  • Used tree pruning (uppper/lower bounds) and promising computations to analyze whether a partial solution is promising and worth continuing
  • Dynamic memory allocation with new and delete
  • Custom map implementation with templating and using Pair types
